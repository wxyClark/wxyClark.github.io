# AI术语表

## A

### Agent（智能代理）
能够自主感知环境、做出决策并执行动作以实现特定目标的系统。在AI领域，通常指具有自主决策能力的AI系统。

### API（应用程序编程接口）
不同软件系统之间进行交互的规则和接口，允许开发者调用其他系统的功能。

### ASR（自动语音识别）
将 spoken language（口语）转换为 written text（书面文本）的技术。

## B

### BERT（Bidirectional Encoder Representations from Transformers）
一种基于Transformer架构的预训练语言模型，能够双向理解文本语义。

### BPE（Byte Pair Encoding）
一种用于文本分词的算法，通过迭代合并最常见的字符对来构建词汇表。

## C

### CSP（内容安全策略）
一种安全机制，用于防止跨站脚本攻击和其他代码注入攻击。

### CV（计算机视觉）
使计算机能够理解和处理图像、视频等视觉信息的技术领域。

## D

### Decoder（解码器）
Transformer架构的一部分，用于生成文本序列，如GPT系列模型主要使用解码器架构。

### Diffusion Model（扩散模型）
一种生成式AI模型，通过逐步去噪过程生成高质量图像。

### Document Embedding（文档嵌入）
将整个文档转换为固定长度向量的过程，用于文档检索和相似度计算。

## E

### Embedding（嵌入）
将离散数据（如单词、句子、文档）转换为连续向量空间中的向量表示的过程。

### Encoder（编码器）
Transformer架构的一部分，用于理解输入文本，如BERT模型主要使用编码器架构。

### ETL（抽取、转换、加载）
数据处理的三个步骤：从源系统抽取数据，转换为目标格式，加载到目标系统。

## F

### Fine-tuning（微调）
在预训练模型的基础上，使用特定领域的数据进一步训练模型，以提高其在特定任务上的性能。

### Function Calling（函数调用）
大语言模型调用外部函数或工具的能力，用于获取实时信息或执行特定操作。

## G

### GPT（Generative Pre-trained Transformer）
由OpenAI开发的一系列生成式预训练Transformer模型，如GPT-3、GPT-4。

### GPU（图形处理器）
用于加速图形渲染和并行计算的硬件，广泛用于AI模型训练和推理。

## H

### Hallucination（幻觉）
大语言模型生成的内容与事实不符，但模型却自信地将其呈现为真实信息的现象。

### Hugging Face
一个开源AI平台，提供各种预训练模型和工具，用于NLP、CV等领域。

## I

### In-context Learning（上下文学习）
大语言模型通过在提示中提供示例，无需微调即可学习执行特定任务的能力。

### Intent Recognition（意图识别）
识别用户输入或请求的意图或目的的技术，常用于对话系统。

## J

### Jupyter Notebook
一种交互式计算环境，允许在单个文档中组合代码、文本、图表等。

## K

### Knowledge Graph（知识图谱）
一种结构化的知识表示方式，通过节点和边来表示实体及其关系。

### KNN（K-最近邻算法）
一种用于分类和回归的机器学习算法，通过查找最近的K个样本进行预测。

## L

### LLM（大语言模型）
参数量巨大、训练数据海量的深度学习模型，能够理解和生成人类语言。

### Lora（Low-Rank Adaptation）
一种高效的模型微调技术，通过训练低秩矩阵来适应特定任务，而不是微调整个模型。

## M

### ML（机器学习）
AI的一个分支，使计算机能够从数据中学习，而无需显式编程。

### MLOps（机器学习运维）
将DevOps原则应用于机器学习，涵盖模型开发、训练、部署和监控的整个生命周期。

### Multi-modal（多模态）
能够处理和理解多种数据模态（如文本、图像、音频）的AI系统。

## N

### NER（命名实体识别）
识别文本中具有特定意义的实体（如人名、地名、组织名）的技术。

### NLP（自然语言处理）
使计算机能够理解、处理和生成人类语言的技术领域。

### NLU（自然语言理解）
NLP的一个分支，专注于使计算机理解自然语言的语义和意图。

### NLG（自然语言生成）
NLP的一个分支，专注于使计算机生成自然、流畅的人类语言。

## O

### OCR（光学字符识别）
将图像中的文字转换为可编辑文本的技术。

### OpenAI
一家人工智能研究实验室，开发了GPT系列、DALL-E等知名AI模型。

## P

### Prompt Engineering（提示工程）
设计和优化提示词，以引导大语言模型生成期望输出的技术。

### Python
一种广泛用于AI和数据科学的编程语言，以其简洁性和丰富的库生态而闻名。

## Q

### QA（问答系统）
能够回答用户提出的自然语言问题的AI系统。

### Q-learning
一种强化学习算法，通过学习动作值函数来优化决策策略。

## R

### RAG（检索增强生成）
结合检索和生成的AI技术，通过从外部知识库检索相关信息，增强大语言模型的生成能力。

### ReAct（Reasoning and Acting）
一种Agent架构，结合了推理和动作执行，允许模型通过思考和行动来解决问题。

### Reinforcement Learning（强化学习）
一种机器学习范式，智能体通过与环境交互，学习最优策略以最大化奖励。

### Retrieval（检索）
从大量数据中查找和获取相关信息的过程，如向量检索、关键词检索。

### RLHF（人类反馈强化学习）
一种训练大语言模型的技术，通过人类反馈来对齐模型输出与人类偏好。

## S

### Semantic Search（语义搜索）
基于文本语义理解的搜索技术，而不仅仅是关键词匹配。

### Sentiment Analysis（情感分析）
识别和提取文本中情感倾向（如正面、负面、中性）的技术。

### SFT（监督微调）
使用标注数据对预训练模型进行微调的过程。

### Speech Synthesis（语音合成）
将文本转换为自然语音的技术，也称为TTS（Text-to-Speech）。

### SQL（结构化查询语言）
用于管理关系型数据库的标准语言，用于查询、插入、更新和删除数据。

### SVM（支持向量机）
一种监督学习算法，用于分类和回归任务，通过找到最优超平面来分隔不同类别。

### Swarm Intelligence（群体智能）
多个简单个体通过相互协作，表现出复杂智能行为的现象，如蚁群算法、粒子群优化。

## T

### Tokenization（分词）
将文本分割为更小的单位（如单词、子词）的过程，是NLP处理的第一步。

### Transformer
一种基于自注意力机制的深度学习架构，广泛用于NLP、CV等领域，是现代大语言模型的基础。

### TS（TypeScript）
JavaScript的超集，添加了静态类型，用于开发大型Web应用。

## U

### UI（用户界面）
用户与计算机系统交互的界面，包括图形界面、命令行界面等。

### UX（用户体验）
用户使用产品或系统时的整体体验和感受。

## V

### Vector Database（向量数据库）
专门用于存储和检索向量数据的数据库，能够高效处理高维向量的相似度搜索。

### Vector Embedding（向量嵌入）
将文本、图像等数据转换为高维向量表示的过程，用于机器学习和深度学习模型。

### Vision Transformer（ViT）
将Transformer架构应用于计算机视觉任务的模型，能够处理图像数据。

## W

### Webhook
一种HTTP回调，用于实现系统间的实时通信，当特定事件发生时触发。

### Workflow（工作流）
将一系列任务按照预定义的规则和顺序组织起来，实现自动化执行的过程。

## X

### XAI（可解释人工智能）
使AI系统的决策过程可解释、可理解的技术，用于提高AI系统的透明度和可信度。

### XML（可扩展标记语言）
一种用于存储和传输数据的标记语言，具有自我描述性。

## Y

### YOLO（You Only Look Once）
一种实时目标检测算法，能够在单次前向传播中检测图像中的多个目标。

## Z

### Zero-shot Learning（零样本学习）
模型能够处理从未见过的类别或任务，无需额外训练数据的能力。

### ZK-SNARK（零知识证明）
一种密码学技术，允许一方在不泄露任何额外信息的情况下，向另一方证明某个声明是真实的。

# 技术栈术语表

## A

### Apache Airflow
一个开源的工作流管理平台，用于编排和调度复杂的数据流水线。

### AWS（亚马逊网络服务）
亚马逊提供的云 computing 平台，包括计算、存储、数据库等服务。

## C

### Chrome
Google开发的Web浏览器，基于Chromium开源项目。

### Docker
一种容器化平台，用于打包、分发和运行应用程序及其依赖。

## G

### GitHub
一个基于Git的代码托管平台，用于版本控制和协作开发。

### GitLab
一个基于Git的DevOps平台，提供代码托管、CI/CD、项目管理等功能。

## K

### Kubernetes
一个开源的容器编排平台，用于自动化部署、扩展和管理容器化应用。

## L

### LangChain
一个用于构建基于LLM的应用程序的框架，提供了链接不同组件的工具和接口。

## M

### MongoDB
一个开源的文档型NoSQL数据库，用于存储和检索半结构化数据。

### MySQL
一个开源的关系型数据库管理系统，广泛用于Web应用和其他场景。

## N

### Node.js
一个基于Chrome V8引擎的JavaScript运行时，用于开发服务器端应用。

### n8n
一个开源的自动化平台，用于创建可视化工作流，支持多种集成。

## P

### PostgreSQL
一个开源的关系型数据库管理系统，具有强大的功能和可扩展性。

### Prefect
一个现代化的工作流管理系统，用于构建、调度和监控数据工作流。

## R

### React
一个用于构建用户界面的JavaScript库，由Facebook开发。

### Redis
一个开源的内存数据库，用于缓存、消息队列、实时分析等场景。

### Rust
一种系统编程语言，注重安全性、并发性和性能。

## S

### SQLite
一个轻量级的嵌入式关系型数据库，无需单独的服务器进程。

### SQLAlchemy
一个Python SQL工具包和ORM框架，用于与关系型数据库交互。

## T

### TensorFlow
Google开发的开源机器学习框架，用于构建和训练各种AI模型。

### TypeScript
JavaScript的超集，添加了静态类型，用于开发大型Web应用。

## V

### Vim
一个高效的文本编辑器，广泛用于编程和系统管理。

### VS Code
Microsoft开发的开源代码编辑器，支持多种编程语言和扩展。

## Z

### Zapier
一个商业自动化平台，用于连接不同的Web应用，无需编码。
