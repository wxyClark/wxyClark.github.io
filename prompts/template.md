# Quartz & Obsidian 页面模板提示词

## 模板说明

本提示词用于在基于 Quartz 与 Obsidian 构建的 GitHub Pages 项目中创建标准化的页面模板。以下包含 8 种常见页面类型的模板定义，每种模板均包含必要的元数据字段、标准内容结构、排版规范和可选扩展模块。

## 1. 个人博客模板

### 元数据字段
- **title**: 博客标题（必填）
- **date**: 发布日期（必填，格式：YYYY-MM-DD）
- **tags**: 标签列表（必填，数组格式）
- **categories**: 分类（必填，字符串）
- **summary**: 摘要（可选，简短描述）
- **author**: 作者（可选，默认值）
- **slug**: 自定义链接（可选，默认使用标题）

### 内容结构
1. **标题**：使用一级标题
2. **摘要**：简短介绍文章主题
3. **正文**：分节内容，使用二级/三级标题
4. **结论**：总结要点
5. **相关链接**：推荐阅读或参考资料

### 排版规范
- 段落间距：一行空行
- 代码块：使用三个反引号
- 图片：使用标准 Markdown 格式，添加 alt 文本
- 引用：使用 > 符号

### 扩展模块
- **系列文章**：相关文章链接
- **评论区**：Disqus 或其他评论系统
- **分享按钮**：社交媒体分享

## 2. 项目文档模板

### 元数据字段
- **title**: 文档标题（必填）
- **date**: 创建日期（必填，格式：YYYY-MM-DD）
- **tags**: 标签列表（必填，数组格式）
- **categories**: 分类（必填，字符串，建议："Documentation"）
- **version**: 版本号（必填，语义化版本）
- **status**: 状态（可选，如："Draft", "Published", "Deprecated"）
- **maintainer**: 维护者（可选）

### 内容结构
1. **概述**：项目简介和目标
2. **安装指南**：环境要求和安装步骤
3. **使用方法**：基本操作和示例
4. **配置选项**：可配置参数说明
5. **常见问题**：FAQ 部分
6. **贡献指南**：如何参与项目

### 排版规范
- 步骤编号：使用有序列表
- 命令示例：使用代码块
- 配置示例：使用代码块或表格
- 警告信息：使用 > [!WARNING] 格式（Obsidian 提示框）

### 扩展模块
- **API 参考**：详细接口说明
- **变更日志**：版本更新记录
- **路线图**：未来计划

## 3. 学习笔记模板

### 元数据字段
- **title**: 笔记标题（必填）
- **date**: 创建日期（必填，格式：YYYY-MM-DD）
- **tags**: 标签列表（必填，数组格式）
- **categories**: 分类（必填，字符串，建议："Notes"）
- **course**: 课程名称（可选）
- **instructor**: 讲师（可选）
- **priority**: 优先级（可选，如："High", "Medium", "Low"）

### 内容结构
1. **学习目标**：本次学习的重点
2. **核心概念**：关键知识点
3. **示例代码**：实践示例
4. **疑问点**：未理解的内容
5. **总结**：学习收获
6. **复习计划**：后续学习安排

### 排版规范
- 概念定义：使用 **加粗** 或 > [!NOTE] 格式
- 代码示例：使用代码块
- 列表：使用无序列表
- 表格：用于对比或整理信息

### 扩展模块
- **参考资料**：书籍、文章链接
- **练习题目**：巩固练习
- **思维导图**：知识结构可视化

## 4. 知识库条目模板

### 元数据字段
- **title**: 条目标题（必填）
- **date**: 创建日期（必填，格式：YYYY-MM-DD）
- **tags**: 标签列表（必填，数组格式）
- **categories**: 分类（必填，字符串，建议："Knowledge Base"）
- **aliases**: 别名（可选，数组格式）
- **related**: 相关条目（可选，数组格式）
- **status**: 状态（可选，如："Active", "In Review", "Archived"）

### 内容结构
1. **定义**：条目核心概念
2. **背景**：历史和上下文
3. **应用场景**：使用情况
4. **相关概念**：关联知识
5. **参考资料**：来源链接

### 排版规范
- 定义：使用 > [!DEFINITION] 格式
- 层级结构：使用标题层级
- 链接：使用 Obsidian 内部链接格式
- 表格：用于结构化数据

### 扩展模块
- **术语表**：相关术语解释
- **案例研究**：实际应用例子
- **常见误区**：容易混淆的点

## 5. 教程指南模板

### 元数据字段
- **title**: 教程标题（必填）
- **date**: 创建日期（必填，格式：YYYY-MM-DD）
- **tags**: 标签列表（必填，数组格式）
- **categories**: 分类（必填，字符串，建议："Tutorial"）
- **difficulty**: 难度（必填，如："Beginner", "Intermediate", "Advanced"）
- **estimated_time**: 预计时间（可选，如："30 minutes"）
- **prerequisites**: 前置条件（可选，数组格式）

### 内容结构
1. **简介**：教程目标和预期成果
2. **准备工作**：所需工具和环境
3. **步骤**：详细操作步骤
4. **验证**：验证结果的方法
5. **故障排除**：常见问题解决
6. **进阶内容**：扩展学习方向

### 排版规范
- 步骤：使用有序列表，每步配小标题
- 代码：使用代码块，添加语言标识
- 图片：使用标准 Markdown 格式，添加说明
- 提示：使用 > [!TIP] 格式

### 扩展模块
- **视频教程**：配套视频链接
- **练习项目**：实践任务
- **资源下载**：相关文件下载

## 6. 资源清单模板

### 元数据字段
- **title**: 清单标题（必填）
- **date**: 创建日期（必填，格式：YYYY-MM-DD）
- **tags**: 标签列表（必填，数组格式）
- **categories**: 分类（必填，字符串，建议："Resources"）
- **last_updated**: 最后更新日期（可选，格式：YYYY-MM-DD）
- **source**: 来源（可选）
- **purpose**: 用途（可选）

### 内容结构
1. **简介**：清单目的和范围
2. **分类**：资源分类
3. **资源列表**：详细条目，包含名称、描述、链接
4. **使用建议**：如何有效利用
5. **更新记录**：变更历史

### 排版规范
- 资源条目：使用表格或列表
- 链接：使用标准 Markdown 链接格式
- 分类：使用二级标题
- 描述：简洁明了

### 扩展模块
- **评分系统**：资源质量评估
- **使用案例**：资源应用示例
- **贡献指南**：如何添加新资源

## 7. 会议记录模板

### 元数据字段
- **title**: 会议标题（必填）
- **date**: 会议日期（必填，格式：YYYY-MM-DD）
- **tags**: 标签列表（必填，数组格式）
- **categories**: 分类（必填，字符串，建议："Meeting Notes"）
- **time**: 会议时间（可选，如："14:00-15:30"）
- **attendees**: 参会人员（可选，数组格式）
- **location**: 会议地点（可选）

### 内容结构
1. **会议信息**：时间、地点、参会人员
2. **议程**：讨论主题
3. **讨论内容**：详细讨论记录
4. **决策事项**：达成的决议
5. **行动项**：任务分配和截止日期
6. **下次会议**：时间和议程预告

### 排版规范
- 议程项目：使用无序列表
- 行动项：使用表格，包含任务、负责人、截止日期
- 时间戳：使用 **加粗** 格式
- 引用：使用 > 符号

### 扩展模块
- **会议材料**：相关文档链接
- **录音链接**：会议录音（如适用）
- **会议纪要**：简明摘要

## 8. 每日日志模板

### 元数据字段
- **title**: 日志标题（必填，建议："YYYY-MM-DD 日志"）
- **date**: 日期（必填，格式：YYYY-MM-DD）
- **tags**: 标签列表（必填，数组格式）
- **categories**: 分类（必填，字符串，建议："Daily Log"）
- **mood**: 心情（可选，如："Good", "Neutral", "Tired"）
- **weather**: 天气（可选）

### 内容结构
1. **今日完成**：已完成的任务
2. **明日计划**：待完成的任务
3. **遇到的问题**：挑战和困难
4. **学习收获**：新知识和技能
5. **反思**：思考和感悟
6. **其他**：杂项记录

### 排版规范
- 任务列表：使用勾选框（- [ ]）
- 时间线：使用无序列表
- 重点内容：使用 **加粗** 或 > [!NOTE] 格式
- 简短记录：保持简洁

### 扩展模块
- **习惯追踪**：日常习惯记录
- **健康状况**：身体状况记录
- **成就记录**：小成就和进步

## 验证文档创建指令

为验证上述模板的效果和页面渲染质量，请按照以下步骤操作：

1. 在项目的 `content` 目录下创建 `demo` 子目录：
   ```bash
   mkdir -p content/demo
   ```

2. 在 `demo` 目录中使用上述模板各生成一个示例 Markdown 文档：
   - `content/demo/blog-example.md`：个人博客示例
   - `content/demo/docs-example.md`：项目文档示例
   - `content/demo/notes-example.md`：学习笔记示例
   - `content/demo/kb-example.md`：知识库条目示例
   - `content/demo/tutorial-example.md`：教程指南示例
   - `content/demo/resources-example.md`：资源清单示例
   - `content/demo/meeting-example.md`：会议记录示例
   - `content/demo/daily-example.md`：每日日志示例

3. 每个示例文档应包含：
   - 完整的元数据字段
   - 标准的内容结构
   - 符合排版规范的内容
   - 至少一个扩展模块

4. 构建并部署项目，验证页面渲染效果：
   ```bash
   npx quartz build
   # 部署到 GitHub Pages
   ```

5. 检查每个示例页面的：
   - 元数据显示是否正确
   - 内容结构是否清晰
   - 排版格式是否规范
   - 扩展模块是否正常显示
   - 页面链接是否有效

通过以上步骤，可以确保模板的可用性和渲染质量，为后续的内容创作提供标准化的框架。
