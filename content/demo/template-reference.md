# Quartz & Obsidian 文档模板参考

## 模板说明

本文件是基于 Quartz 与 Obsidian 构建的 GitHub Pages 项目的标准化文档模板参考。包含 8 种常见页面类型的模板结构，每种模板均包含必要的元数据字段、标准内容结构、排版规范和可选扩展模块。

## 1. 个人博客模板

### 元数据字段
```yaml
title: "博客标题"
date: "YYYY-MM-DD"
tags: [标签1, 标签2, 标签3]
categories: "分类"
summary: "简短摘要"
author: "作者"
slug: "自定义链接"
```

### 内容结构
```markdown
# 博客标题

## 摘要
简短介绍文章主题...

## 正文第一节
内容...

## 正文第二节
内容...

## 结论
总结要点...

## 相关链接
- [链接1](URL)
- [链接2](URL)
```

### 扩展模块
- **系列文章**：相关文章链接
- **评论区**：Disqus 或其他评论系统
- **分享按钮**：社交媒体分享

## 使用说明

1. **选择模板**：根据内容类型选择合适的模板
2. **填充元数据**：修改 YAML 前置元数据中的占位符
3. **编写内容**：按照模板结构填充实际内容
4. **添加扩展**：根据需要添加相应的扩展模块
5. **验证渲染**：使用 `npx quartz build` 构建项目并检查渲染效果

## 维护建议

- 定期更新模板以适应项目需求变化
- 保持模板结构的一致性和可扩展性
- 在使用过程中收集反馈并持续改进
- 为新的内容类型添加相应的模板

通过使用这些标准化模板，可以确保项目文档的一致性和专业性，提高内容创作效率和页面渲染质量。
